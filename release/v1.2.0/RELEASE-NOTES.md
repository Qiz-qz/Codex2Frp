# Codex2Frp v1.2.0

This public release updates Codex2Frp to the latest backend control fixes from the current development line while keeping the public version series on `v1.x`.

## Changes

- Model, reasoning, and speed switches now save the requested mode while Codex is replying, instead of failing the mobile action.
- Minimized Codex Desktop windows are restored before reading the composer plus menu, focusing the composer, switching threads, or sending a stop command.
- Mobile clients no longer treat backend control errors as LAN or remote-link connection failures.
- The Windows control panel remains responsive during status refreshes, log reads, remote-link checks, and longer Codex-control actions.
- Thread status and current desktop-thread metadata are exposed for realtime mobile synchronization.
